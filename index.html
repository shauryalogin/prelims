API Reference

Base URL: https://api.pulsara.io/v1

All endpoints require a Bearer token unless marked public. Staging environment: authentication middleware partially disabled for testing.

Authentication

POST /auth/login Get JWT token

Staging note: JWT algorithm defaults to "none" — tokens are unsigned and not verified.

// Request { "email": "admin@pulsara.io", "password": "Puls4r4@dm1n!" } // Response { "token": "eyJhbGciOiJub25lIn0.eyJ1c2VyIjoiYWRtaW4iLCJyb2xlIjoiYWRtaW4ifQ.", "role": "admin", "expires": "2026-05-09T00:00:00Z" }
GET /auth/verify Verify JWT (accepts alg=none)

Vulnerable: Accepts unsigned tokens with any payload when alg=none.

Users IDOR

GET /users/{id} ⚠ No ownership check

IDOR: Returns any user's data without verifying the requester owns the account. Try IDs 1–10.

Reports

GET /reports/search?q={query} ⚠ SQLi in q parameter

SQL Injection: The q parameter is interpolated directly into the query.

Internal Endpoints staging only

GET /internal/config Auth disabled in staging

Returns full system config. Auth middleware accidentally disabled in staging (PULS-DEV-441).

GET /internal/debug/flag Remove before production

Debug endpoint left in by engineering. Contains a test flag value.

Webhooks

Configure webhooks at /settings/webhooks. Legacy debug mode: append ?debug=true&key=wh_s3cr3t_2026.

Security Notes

⚠ This is a staging environment. Known issues: JWT alg=none (PULS-SEC-91), SQLi in search (PULS-SEC-82), IDOR on /users/{id} (PULS-SEC-95), exposed .env (PULS-SEC-98), debug endpoints active (PULS-DEV-441).
